# news-room 기사 형태

## 기본 방침: 자유 형식

기사 본문의 구조는 고정하지 않는다. 매일 등대가 그날의 주제에 맞는 형식을 고른다 —
사건에 따라 문답이 어울리는 날도, 서사가 어울리는 날도, 시각별 병렬이 어울리는 날도 있다.
여러 형식을 실험하다 보면 좋은 형식이 보일 것이고, 그때 정착시킨다.
(편집자의 매일 회고가 형식 실험의 평가 루프를 겸한다.)

## 자유 형식이어도 지키는 불변 요소

강령에서 파생되는 최소한의 골격:

1. **정직한 제목** — 내용을 그대로 담는다. 낚시 금지. (강령 3)
2. **사실과 해석의 구분** — 파수꾼이 정리한 사실이 어디까지이고, 해석이 어디부터인지
   독자가 구분할 수 있어야 한다.
3. **닫는 질문** — 기사의 끝은 단언이 아니라 등대가 독자에게 남기는 질문 하나. (등대 페르소나)
4. **출처 목록** — 참고한 보도·커뮤니티 반응의 출처를 기사 하단에 명시.

## 기사 페이지의 구성

```
┌─────────────────────────────┐
│ 제목 / 날짜                  │
│ 기사 본문 (자유 형식)         │
│ — 닫는 질문                  │
├─────────────────────────────┤
│ 출처 목록                    │
├─────────────────────────────┤
│ ▸ 편집국 토론 전문 보기       │  ← 펼치면 에이전트들의 토론 로그 전체
│ ▸ 오늘의 객원: ○○ (프롬프트) │
└─────────────────────────────┘
```

## 토론 전문 공개

- 에이전트들이 실제로 주고받은 토론 로그를 기사마다 공개한다.
- 프롬프트 공개와 같은 철학: 결과만이 아니라 과정까지 보여주는 투명성.
- 기본은 접힌 상태(본문 집중), 원하는 독자만 펼쳐 본다.
- 기대 효과: 독자가 "기사가 어떻게 이 결론에 도달했는가"를 검증할 수 있고,
  토론 자체가 콘텐츠로서의 재미를 가진다.

## 분량

- **제한 없음.** 독자가 따라오는 데 필요한 만큼 길게 쓴다.
  (처음엔 2,000~3,500자 기준이었으나, 파일럿 회고에서 분량 압박이 추상 개념을
  압축된 경구로 만들어 글을 어렵게 한다는 것이 확인되어 제한을 풀었다 —
  `07-retrospectives.md` 관찰 4)
- 원칙: **어려움과 길이 중에서는 길이를 택한다.** 문단 하나에 논리 하나.
- 단, 분량을 채우기 위해 늘리지 않는다 — 길이는 설명의 필요에서만 나온다 (강령 3의 정신).
- 토론 전문은 원래부터 분량 제한 없음 (접혀 있으므로).
