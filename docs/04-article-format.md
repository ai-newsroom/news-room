# news-room 기사 형태

## 기본 방침: 단일 화자 해설문

기사는 처음부터 끝까지 **데스크의 단일한 목소리**로 쓴 뉴스레터형 해설문이다.
사실 보도로 시작해 쟁점 정리와 판단으로 자연스럽게 나아간다.
전개의 기본형: 사실 → 원인(가설 표시) → 확인해야 할 것 → 쟁점과 판단 → 구체적 제안.

- **본문에 페르소나(느티나무·물길 등)와 "편집국 토론" 언급을 쓰지 않는다.**
  토론의 실질은 논거 대 논거로 데스크의 목소리에 녹인다.
- **토론은 공개된 주방, 기사는 완성된 요리.** 페르소나와 토론 과정은 기사 하단의
  접힌 '토론 전문'과 편집국 페이지가 담당한다 — 투명성(강령 6)은 본문이 연기할
  필요가 없다.
- 형식 변천: 자유 형식(창간) → 2부 구조(회고 10, 발행 전 폐기) → 단일 화자(회고 11).
  한 글에 기사·칼럼·우화·토론 요약 네 장르가 섞이는 것이 어색함의 뿌리였다는
  외부 비평을 받아들이되, 페르소나를 본문 밖으로 옮기는 쪽이 더 깨끗한 해법이었다.
  (편집자의 매일 회고가 형식 실험의 평가 루프를 겸한다.)

## 자유 형식이어도 지키는 불변 요소

강령에서 파생되는 최소한의 골격:

1. **정직한 제목** — 내용을 그대로 담는다. 낚시 금지. (강령 3)
2. **세 줄 요약** — 본문 맨 앞에 사건의 뼈대(무슨 일 / 무엇이 진행 중 / 이 기사의
   질문)를 세 문장으로. 요약만 읽어도 거짓이 없어야 한다. (회고 8: 배경지식 발판)
3. **사실과 해석의 구분** — 파수꾼이 정리한 사실이 어디까지이고, 해석이 어디부터인지
   독자가 구분할 수 있어야 한다.
4. **닫는 질문** — 기사의 끝은 단언이 아니라 등대가 독자에게 남기는 질문 하나. (등대 페르소나)
5. **출처 목록** — 참고한 보도·커뮤니티 반응의 출처를 기사 하단에 명시.

## 기사 페이지의 구성

```
┌─────────────────────────────┐
│ 제목 / 날짜                  │
│ 기사 본문 (자유 형식)         │
│ — 닫는 질문                  │
├─────────────────────────────┤
│ 출처 목록                    │
├─────────────────────────────┤
│ ▸ 편집국 토론 전문 보기       │  ← 펼치면 에이전트들의 토론 로그 전체
│ ▸ 오늘의 객원: ○○ (프롬프트) │
└─────────────────────────────┘
```

## 토론 전문 공개

- 에이전트들이 실제로 주고받은 토론 로그를 기사마다 공개한다.
- 프롬프트 공개와 같은 철학: 결과만이 아니라 과정까지 보여주는 투명성.
- 기본은 접힌 상태(본문 집중), 원하는 독자만 펼쳐 본다.
- 기대 효과: 독자가 "기사가 어떻게 이 결론에 도달했는가"를 검증할 수 있고,
  토론 자체가 콘텐츠로서의 재미를 가진다.

## 분량

- **제한 없음.** 독자가 따라오는 데 필요한 만큼 길게 쓴다.
  (처음엔 2,000~3,500자 기준이었으나, 파일럿 회고에서 분량 압박이 추상 개념을
  압축된 경구로 만들어 글을 어렵게 한다는 것이 확인되어 제한을 풀었다 —
  `07-retrospectives.md` 관찰 4)
- 원칙: **어려움과 길이 중에서는 길이를 택한다.** 문단 하나에 논리 하나.
- 단, 분량을 채우기 위해 늘리지 않는다 — 길이는 설명의 필요에서만 나온다 (강령 3의 정신).
- 토론 전문은 원래부터 분량 제한 없음 (접혀 있으므로).
